# Contributing to PyClaw

Thanks for taking the time to contribute.

## Getting started

1. Fork the repository
2. Create a feature branch off `main`
3. Make your changes
4. Open a pull request with a clear description of what you changed and why

## Code style

Try to keep code readable and consistent with the existing style. Running `pylint` or `flake8` before submitting helps catch obvious issues, but it is not a hard requirement for now.

## Tests

If you are adding new functionality, a `pytest` test for it is appreciated. The project does not have full test coverage yet, so any tests you add are a net positive.

## Reporting bugs

Open an issue. Describe what you did, what you expected, and what actually happened. Include your OS and Python version.

## Asking questions or discussing ideas

Use Issues for specific bugs or feature requests. Use Discussions for broader questions, ideas, or anything that does not fit a bug report.

## What to work on

Check the open issues and milestones for things that need doing. If you want to work on something not listed, open a discussion first so we can talk through the approach before you put in the effort.
